# 名称 ： SkillNet / Tech Relay
**〜 業務の「副産物」を「資産」に変え、対話で知恵を繋ぐ技術継承プラットフォーム 〜**
## サービス概要
SkillNet は、エンジニアの日常（メール・書類・チャット）からAIが自動で技術的エッセンスを抽出集積し、受け継ぐ側には対話を通じてピンポイントで答えを提示する、ナレッジ循環システムで、以下のような問題を解決します。

1. 「技術継承が必要なのはわかっているが、継承資料を作るのは面倒」
2. 「たくさんの資料はあるが、今知りたいことがどこにあるかわからない」
3. 「継ぐべき技術が熟練者と未熟者の間で噛み合わず結果的に継承できない」

## このサービスへの思い
あらゆるエンジニアリングにおいて熟練者と未熟者の二極化が進行する昨今、技術継承は正に部門の持続的成長を成す重要課題です。しかし継承行為はその時々の業績には直結しないため軽視されがちで、日々後回しとなり、結果的に全く何も継承できていないことが散見されます。

このサービスの活用によりエンジニアリングの技術レベルの底上げと持続的継承の一助になることを願っています。

## ユーザー層
技術継承が必要とされる部門のエンジニアの方々

## サービスの利用イメージ
### 1. 💡 現場での意思決定を「そのまま」継承資産に
>シーン :
- ベテランエンジニアのAさんが、外部委託先へ「なぜこの工法を選んだか」の指示メールを送る。
>アクション:
- Aさんは、メールのBCCにentry@skillnet-relay.jp を入れるだけ。
>結果:
- アプリがメールを解析。半年後に別の現場で「なぜあの時この工法にしたんだっけ？」と誰かが疑問に思った際、この時の決定理由が「技術資産」として即座に検索可能になります。

### 2. 💦 手が離せない時の「インタビュー継承」
>シーン: 
- 引退間近の熟練職人Bさんのノウハウを残したいが、Bさんは文書作成が苦手。
>アクション: 
- Bさんが移動中にアプリの「Q&A Relay」を起動。AIが「〇〇の調整で、一番気を使っている感覚は？」と問いかけ、Bさんはそれに回答する。（音声入力）。
>結果: 
- AIが回答を整理し、プロの「コツ」を構造化したマニュアルに自動生成。若手がいつでも「熟練の感覚」を読み解けるようになります。

### 3. 🐏 迷える若手の「AIコンシェルジュ」
>シーン: 
- 若手エンジニアのCさんが、現場で見たことのないエラーに直面。「何が原因か、どう調べればいいかすら分からない」と焦る。
>アクション: 
- Cさんがアプリのチャット欄に「ポンプから変な音がして、圧力が上がらない」と素直に打ち込む。
>結果: 
- AIコンシェルジュが「その音は高い金属音ですか？」と対話で状況を絞り込み、過去10年分のメールや報告書から「5年前のDさんのトラブル対応記録」を特定して提示。解決までの最短ルートを示します。

## ユーザー獲得について
### 🚚 導入・展開イメージ (Deployment & Integration)

#### ☁️ クラウド構成
AWS上に Docker コンテナとしてデプロイ

#### 🔗 既存ツールとの接続
ユーザーが新しいツールを「使い始める」コストを最小化するため、以下のインターフェースを提供します。
* **SkillNet Mailbox:** 専用の受信用アドレスを提供。既存のメーラー（Outlook/Gmail等）のBCCに入れるだけで導入完了。
* **SkillNet Connect:** Slack/Teams との Webhook 連携により、チャットログをボタン一つで資産化。

#### 📱 モバイル対応（将来構想）
現場での「Q&A Relay（音声入力）」をスムーズにするため、PWA(Progressive Web App)に対応。スマートフォンのホーム画面から、現場の「今」を即座に記録できます。
## サービスの差別化ポイント・推しポイント
>「既存のナレッジツールとの違い：ドキュメント作成というタスクを抹消する」
* 従来のツールは『書く時間』を要求しましたが、SkillNet Relayは『業務の痕跡』から価値を抽出します。技術者が本来の業務に集中しながら、組織の知恵が勝手に深まっていく「完全受動型」の継承プラットフォームです。
  - Zero-Effort Ingestion: ユーザーに「資料作成」を強いない、業務への徹底的な「すり込み」。
  - Contextual Spider Web: キーワード一致ではなく、AIによる「文脈のネットワーク化」で関連知識を網羅。
  - Junior-Friendly UX: 検索窓に代わり、AIコンシェルジュが過去の熟練者の思考回路へと導く。

## 機能候補
### 🏃 5つのコア・リレー (Knowledge Relays)
### 1. Email Relay 〈受動的収集〉
- 外部委託先や社内への技術指示・判断メールのBCCに専用アドレスを入れるだけ。
- AIが本文から「何を決めたか（結論）」と「なぜか（根拠）」を構造化して保存。
### 2. Document Relay 〈自動構造化〉
- 技術提案書、計算書、報告書などのファイルをアップロード。
- AIが文書内の主要な技術パラメーターや設計思想を解析し、既存ナレッジと紐付け。
### 3. Chat Relay 〈プロセス要約〉
- Slack等のチャットツールでの試行錯誤を特定スタンプでピックアップ。
- 散らばった投稿を一本の「トラブル解決ストーリー」として要約。
### 4. Q&A Relay 〈対話型継承〉
- 熟練者の頭にある「言語化されていないコツ」をAIがインタビュー。
- カテゴリを選ぶとAIが質問を投げかけ、答えるだけで質の高い継承資料を自動生成。
### 5. Consult Relay 〈逆引きコンシェルジュ〉
- 若手の「何がわからないかわからない」をAIが紐解く。
- 曖昧な悩みをAIが対話で具体化し、4つのリレーで溜まった膨大な資産から「今必要な知恵」をレコメンド。
