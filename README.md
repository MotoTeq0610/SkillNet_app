## サービス概要
SkillNet は熟練者と未熟者の間で経験知見の循環を図り技術継承を促進するサービスです。
1. 熟練エンジニアの日常業務（指示メール・技術書類）からAIが自動で技術情報を抽出集積します。
2. 熟練エンジニアは自らが資料作成をせずともアプリからの質問に答えることで継承資料を作成できます。
3. 受け継ぐ側はアプリとの対話を通じて集積された技術DBからピンポイントで今の業務に欲しいノウハウを取得・活用できます。

## このサービスへの思い・作りたい理由
あらゆるエンジニアリングにおいて熟練者と未熟者の二極化が進行する昨今、技術継承は正に部門の持続的成長を成す重要課題です。

最近、ある装置の制御ソフト開発において業務が進まない若手に、「なぜ進捗が芳しくないのか？」と確認したところ、「ソフト自体は完成したが装置保護のパラメータを決定することができず止まっている」との答えがありました。「過去のリストを参照すれば良いではないか？」と投げかけましたが、「新たな装置が過去のパラメータのままで良いのかどうか判断ができないし、間違って実機を壊してしまうのが怖くて、、、」という返事でした。
ならばベテランにノウハウを聞けば良いのですが、若手曰く「ベテランは忙しくて不在がち、自分が聞くことで邪魔をしたくないし、『そんなことも分からないのか？』と思われたくない」というリアクション。一方でベテラン層も「聞かれなければ応答はしない。常に若手の心配をするほどゆとりは無いし、まして教育資料作りなんてとても、、、」という状況で、言葉を失ってしまいました。

改めて周囲をよく見ると、そんなエピソードは日常的に起きており、業務停滞の要因であるとともに、若手育成・技術継承への大きな障害になっていました。
私自身も、日々の業務に忙殺され教育訓練を後回しにしてしまい、何も継承できていないことを痛感しています。
ですが「時間がない」のは、怠慢ではなく現場のリアルです。だからこそ、「頑張って残す」のではなく「勝手に残る」仕組みを作らなければ、エンジニア業界の知恵や経験知見は途絶えてしまう。その危機感が、このアプリ開発の原動力です。
私自身がこのサービスを早期に立ち上げ、価値を確認した上で広く提供することで、エンジニアリング業界の技術継承の一助になることを切に願っています。

## ユーザー層について
技術継承を考えている熟練エンジニアと経験知見を得ようとする若手エンジニアが存在する組織・グループがターゲットです。

本来、技術継承は手間をかけて綺麗な資料を作って残すのではなく、熟練者の日々の業務から滲み出る経験知見が集積されるべきと考えています。
受け継ぐ側も、資料を探し読み解くのではなく、今知りたいことを直接質問して答えを得られる体験を目指します。

## サービスの利用イメージ
🔳 経験知見の集積時
1. 日々の送信メールのBCCにSkillNet E-mailアドレスを入れて送信する。
2. 普段の業務で作成するドキュメントを指定フォルダに保存する。
3. メールやドキュメント以外の集積は、アプリからの質問に答えるだけ。

🔳 経験知見の参照時（AIコンシェルジュ）
1. 知りたいカテゴリを選択することで、AIが質問を投げかけ、答えるだけで今必要な知見を取得できる。
2. 曖昧な質問をAIが対話で具体化し、集積情報から必要な情報をリコメンドする。

## ユーザーの獲得について
先ずはプロトタイプを私が所属する少人数グループで使用し検証と改良を行います。
また、同様の悩みを抱える親しい同業者にも検証を依頼し、現場のフィードバックを反映させます。
その後、GitHub上での公開、AWS上へのデプロイ（Dockerコンテナ構成）を予定します。

## サービスの差別化ポイント・推しポイント
* 従来のツールは資料などを『書く時間』が必要でしたが、SkillNet は『業務の痕跡』から価値を抽出します。熟練エンジニアが手間を掛けずに、組織の知恵が勝手に深まっていく継承プラットフォームです。
  - ユーザーに「資料作成」を強いない、業務への徹底的な「すり込み」。
  - キーワード一致ではなく、AIの「文脈のネットワーク化」で関連知識を網羅。
  - 検索窓に代わり、AIコンシェルジュが過去の熟練者の思考回路へと導く。

## 機能候補 （MVPリリースと本リリースの区分を含む）
### 🔳 ユーザ認証と管理
1. 【MVP】ユーザ登録 / ログイン
2. 【MVP】プロフィール管理（熟練者/若手、所属グループなどの属性管理）

### 🔳 5つのリレー (Knowledge Relays)
1~3は基盤機能(MVPリリース)、4,5はMVPリリース後の本リリースで追加予定
#### 1. 【MVP】 Email Relay 〈受動的収集〉
- 外部委託先や社内への技術指示・判断メールのBCCに専用アドレスを入れるだけ。
- AIが本文から「何を決めたか（結論）」と「なぜか（根拠）」を構造化して保存。
#### 2. 【MVP】 Q&A Relay 〈対話型継承〉
- 熟練者の頭にある「言語化されていないコツ」をAIがインタビュー。
- カテゴリを選ぶとAIが質問を投げかけ、答えるだけで質の高い継承資料を自動生成。
#### 3. 【MVP】 Consult Relay 〈逆引きコンシェルジュ〉
- 若手の「何がわからないかわからない」をAIが紐解く。
- 曖昧な悩みをAIが対話で具体化し、4つのリレーで溜まった膨大な資産から「今必要な知恵」をレコメンド。
#### 4. 【本リリース】 Document Relay 〈自動構造化〉
- 技術提案書、計算書、報告書などのファイルをアップロード。
- AIが文書内の主要な技術パラメーターや設計思想を解析し、既存ナレッジと紐付け。
#### 5. 【本リリース】Chat Relay 〈プロセス要約〉
- Slack等のチャットツールでの試行錯誤を特定スタンプでピックアップ。
- 散らばった投稿を一本の「トラブル解決ストーリー」として要約。

## 使用する技術スタック
- Backend: Ruby 3.x / Ruby on Rails 7.x
- Frontend: Hotwire (Turbo/Stimulus), Tailwind CSS
- AI/LLM: OpenAI API (GPT-4o) / LangChain
- Email: Action Mailbox
- Storage: Active Storage (AWS S3)
- Analysis: AI-based Vector Search (関連性検索)
