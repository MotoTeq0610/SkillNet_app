## サービス概要
SkillNet は熟練者と未熟者の間で経験知見の循環を図り技術継承を促進するサービスです。
1. 熟練エンジニアの日常業務（指示メール・技術書類）からAIが自動で技術情報を抽出集積します。
2. 熟練エンジニアは自らが資料作成をせずともアプリからの質問に答えることで継承資料を作成できます。
3. 受け継ぐ側はアプリとの対話を通じて集積された技術DBからピンポイントで今の業務に欲しいノウハウを取得し活用できます。

## このサービスへの思い・作りたい理由
あらゆるエンジニアリングにおいて熟練者と未熟者の二極化が進行する昨今、技術継承は正に部門の持続的成長を成す重要課題です。

私自身、設計関係に身を置く中、日々の業務に忙殺され若手への教育訓練を軽視しがちで、ついつい後回しとなり、結果的に全く何も継承できていないことを痛感しています。
このサービスを立上げ、利用いただくことで熟練設計者の負担が少しでも下がり、且つ、若手設計者への持続的な技術継承の一助になることを願っています。

## ユーザー層について
技術継承を考えている熟練エンジニアと経験知見を得ようとする若手エンジニアが存在する組織・グループがターゲット

本来、技術継承は手間をかけて綺麗な資料を作って残すのではなく、熟練者の日々の業務から滲み出る経験知見が集積される仕組みがあるべきと考えています。

受け継ぐ若手エンジニアも、必要な知見を残された資料から探し読み解くのではなく、今知りたい知見を直接的に質問し答えてもらいたいと考えています。

## サービスの利用イメージ
🔳 経験知見の集積時の利用イメージ（AIによる技術情報の抽出と集積）
1. 日々の送信メールのBCCにSkillNet E-mailアドレスを入れて送信する。
2. 普段の業務で作成するドキュメントを指定フォルダに保存する。
3. メールやドキュメント以外の集積は、アプリからの質問に答えるだけ。

🔳 経験知見の参照の利用イメージ（AIによるコンシェルジュサービス）
1. 知りたいカテゴリを選択することで、AIが質問を投げかけ、答えるだけで今必要な知見を取得できる。
2. 曖昧な質問をAIが対話で具体化し、集積情報から必要な情報をリコメンドする。

## サービスの差別化ポイント・推しポイント
* 従来のツールは資料などを『書く時間』が必要でしたが、SkillNet は『業務の痕跡』から価値を抽出します。熟練エンジニアが手間を掛けずに、組織の知恵が勝手に深まっていく継承プラットフォームです。
  - ユーザーに「資料作成」を強いない、業務への徹底的な「すり込み」。
  - キーワード一致ではなく、AIの「文脈のネットワーク化」で関連知識を網羅。
  - 検索窓に代わり、AIコンシェルジュが過去の熟練者の思考回路へと導く。

## 機能候補
### 🏃 5つのコア・リレー (Knowledge Relays)
### 1. Email Relay 〈受動的収集〉
- 外部委託先や社内への技術指示・判断メールのBCCに専用アドレスを入れるだけ。
- AIが本文から「何を決めたか（結論）」と「なぜか（根拠）」を構造化して保存。
### 2. Document Relay 〈自動構造化〉
- 技術提案書、計算書、報告書などのファイルをアップロード。
- AIが文書内の主要な技術パラメーターや設計思想を解析し、既存ナレッジと紐付け。
### 3. Chat Relay 〈プロセス要約〉
- Slack等のチャットツールでの試行錯誤を特定スタンプでピックアップ。
- 散らばった投稿を一本の「トラブル解決ストーリー」として要約。
### 4. Q&A Relay 〈対話型継承〉
- 熟練者の頭にある「言語化されていないコツ」をAIがインタビュー。
- カテゴリを選ぶとAIが質問を投げかけ、答えるだけで質の高い継承資料を自動生成。
### 5. Consult Relay 〈逆引きコンシェルジュ〉
- 若手の「何がわからないかわからない」をAIが紐解く。
- 曖昧な悩みをAIが対話で具体化し、4つのリレーで溜まった膨大な資産から「今必要な知恵」をレコメンド。
